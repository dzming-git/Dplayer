# Host配置更新说明

## 更新时间
2026-03-13

## 更新内容

### 1. 配置文件更新 (config.json)
在配置文件中添加了 `host` 字段,用于指定服务器监听地址:

```json
{
  "host": "0.0.0.0",
  "ports": {
    "main_app": 80,
    "admin_app": 8080
  }
}
```

### 2. 代码修改

#### app.py (主应用)
- 在全局配置中添加了 `HOST` 变量
- 修改了硬编码的 `host='0.0.0.0'` 为 `host=HOST`
- 更新了启动日志信息

```python
# 全局配置
config = load_config()
PORT = config.get('ports', {}).get('main_app', 80)
HOST = config.get('host', '0.0.0.0')

# 启动时使用配置
app.run(host=HOST, port=PORT, debug=True)
```

#### admin_app.py (管理后台)
- 在全局配置中添加了 `HOST` 变量
- 修改了硬编码的 `host='0.0.0.0'` 为 `host=HOST`
- 更新了启动日志信息

```python
# 全局配置
config = load_config()
ADMIN_PORT = config.get('ports', {}).get('admin_app', 8080)
MAIN_APP_PORT = config.get('ports', {}).get('main_app', 80)
HOST = config.get('host', '0.0.0.0')

# 启动时使用配置
app.run(host=HOST, port=ADMIN_PORT, debug=False)
```

## 配置说明

### host字段的可选值

1. **0.0.0.0** (推荐)
   - 允许所有网络接口访问
   - 支持本地访问 (127.0.0.1)
   - 支持局域网访问
   - 支持外部网络访问
   - 适用于生产环境

2. **127.0.0.1**
   - 仅允许本地访问
   - 更安全,但仅用于开发测试
   - 不支持局域网访问

3. **特定IP地址**
   - 只监听指定的网络接口
   - 例如: `192.168.1.100`
   - 适用于多网卡服务器

## 访问地址

### 当前配置 (host: 0.0.0.0)
- **主应用**: http://0.0.0.0:80 或 http://127.0.0.1:80
- **管理后台**: http://0.0.0.0:8080 或 http://127.0.0.1:8080

### 局域网访问
如果host设置为0.0.0.0,局域网内其他设备可以通过以下地址访问:
- **主应用**: http://<服务器IP>:80
- **管理后台**: http://<服务器IP>:8080

例如,服务器IP为192.168.1.100:
- 主应用: http://192.168.1.100:80
- 管理后台: http://192.168.1.100:8080

## 注意事项

1. **防火墙配置**
   - 如果host设置为0.0.0.0,确保防火墙允许相应端口的入站连接
   - Windows防火墙需要添加规则允许端口80和8080

2. **安全考虑**
   - 在生产环境中,建议添加认证机制
   - 管理后台应该有额外的安全措施
   - 考虑使用HTTPS加密传输

3. **端口占用检查**
   - 启动前会自动检查端口是否被占用
   - 如果端口被占用,可以选择终止占用进程

4. **配置热加载**
   - 修改config.json后需要重启应用才能生效
   - 不支持运行时动态修改host配置

## 优势

1. **集中管理** - host和port都在配置文件中管理
2. **灵活部署** - 可以根据不同环境调整配置
3. **易于维护** - 避免代码硬编码
4. **安全性** - 可以轻松限制访问范围
